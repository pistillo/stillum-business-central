# US-10.2.1 – Piano di implementazione: Editor React (TypeScript) per MODULE/COMPONENT

## Obiettivo

Come sviluppatore, voglio un editor di codice React con supporto TypeScript, in modo da scrivere pools, droplets e triggers con produttività.

## Stato (worktree)

- Editor `MODULE/COMPONENT`: implementato tramite Stillum Theia integrato in iframe (protocollo `postMessage`).
- Load/save sorgenti: implementato via `GET /artifacts/{moduleId}/workspace` e `updateVersion(files)` mediato dal portale.
- Gestione dipendenze npm: implementata tramite `DependenciesPanel` aggiornando `package.json` nella mappa `files` della versione; lookup pacchetti via Runtime Gateway (proxy Nexus).
- Wizard “Nuovo Pool / Droplet / Trigger”: non implementato.

## Architettura (worktree)

- UI: la pagina editor usa Theia per `MODULE/COMPONENT` e mantiene Monaco/StillumForms per gli altri tipi.
- Backend: la Registry salva i file delle versioni su MinIO/S3 con chiavi convenzionali e li espone in `ArtifactVersionResponse.files`.
- Dipendenze npm: la “source of truth” è `package.json` nel set di file; il Publisher estrae `dependencies` e invoca `npm-build-service` in fase di publish.

## Attività residue

1. Definire e implementare il wizard di creazione “Pool/Droplet/Trigger” (template e metadati).
2. Aggiungere test end-to-end per flusso completo (create → edit → dependencies → publish → package su Nexus).
3. Hardening sicurezza build (SCA/audit) e governance dipendenze (policy).
